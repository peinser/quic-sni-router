HOT RELOAD:

This chart's ConfigMap is rendered from .Values.config — `helm upgrade` writes
a new ConfigMap, kubelet propagates it into each pod (typically <60s), and the
router's inotify watch atomically swaps in the new routes. No pod restart, no
in-flight session loss for routes whose backend was preserved (hard cutover
only for routes whose backend disappeared).

To force a pod rollout instead of in-place reload (drops live sessions):

  helm upgrade {{ .Release.Name }} ... --set configMap.checksumAnnotation=true

QUIC versions accepted: v1 (RFC 9000) and v2 (RFC 9369). Other versions are
silently dropped at the version check; the client will fall back via version
negotiation.

WAN-FACING DEPLOYMENT CHECKLIST (see docs/threat-model.md):

  - Backends should be on a non-routable / cluster-internal network.
  - Add upstream BCP 38 / uRPF + per-source rate limiting at the LB.
  - service.externalTrafficPolicy=Local (the chart default) preserves the
    client source IP, which the router uses for session pinning.
